10:T2084,<p>Setelah pada seri sebelumnya (ke-3) kita membuat deployment dengan menggunakan Vercel, di seri terakhir ini (ke-4) kita akan sama-sama belajar bagaimana cara melindungi branch main, karena jika sekarang kamu mencoba melakukan perubahan secara langsung pada branch main, itu bisa dilakukan dengan mudah dan proses CI tidak akan dijalankan, sehingga ada kemungkinan kode yang error ter-deploy oleh Vercel ke production. Maka perlu untuk melindungi branch main sehingga tidak ada yang bisa melakukan perubahan secara langsung pada branch main, termasuk user dengan hak akses <em>Repository admin</em>, yaitu <em>owner</em> (pemilik) repositori tersebut (kamu bisa membaca lebih lanjut mengenai hak akses pada GitHub <a href="https://docs.github.com/en/get-started/learning-about-github/access-permissions-on-github">disini</a>), lalu jika ingin melakukan perubahan harus melalui pull request sehingga proses CI otomatis dijalankan. Dan jika ingin melakukan <em>Merge pull request</em> maka harus lolos proses CI (test-and-lint). Lalu bagaimana cara melindungi branch main? caranya adalah dengan membuat Branch Ruleset.</p>
<blockquote>
<p><strong>Info:</strong> Pastikan kamu sudah mengikuti bagian ke-3. Jika belum, kamu bisa scroll ke paling bawah halaman ini sampai pada bagian Daftar Seri Tulisan dan klik link pada bagian ke-3.</p>
</blockquote>
<p>Ruleset adalah daftar aturan yang berlaku pada repositori, yang dapat membantu kamu untuk mengontrol bagaimana orang berinteraksi dengan branch dan tag di dalam repositori. Yang bisa menggunakan fitur ruleset ini adalah orang dengan hak akses <em>Repository admin</em> atau hak akses <em>Edit repository rules</em>, yang mana orang dengan hak akses ini dapat membuat, edit dan menghapus ruleset untuk sebuah repositori. Kamu juga dapat memiliki hingga 75 ruleset per repositori. Lebih detail mengenai ruleset bisa baca <a href="https://docs.github.com/en/repositories/configuring-branches-and-merges-in-your-repository/managing-rulesets/about-rulesets">disini</a>.</p>
<p>Untuk membuat ruleset, ikuti langkah berikut:</p>
<ol>
<li>Buka remote repositori website counter-js di GitHub kamu, lalu klik <strong>Settings</strong> Â» pada sidebar menu sebelah kiri klik <strong>Rules</strong>, kemudian klik <strong>Rulesets</strong>, pada halaman Rulesets yang terbuka, klik <strong>New ruleset</strong> dan klik <strong>New branch ruleset</strong>:
<img src="/posts/deploy-vercel-github-action/create-branch-ruleset-github.png" alt="create branch ruleset github" width="1366" height="656" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=1366&height=656&loading=lazy&class=mt-6&decoding=async--></li>
<li>Pada bagian <strong>Ruleset Name</strong>, isi dengan nama ruleset</li>
<li>Pada bagian <strong>Enforcement Status</strong>, pilih <strong>Active</strong>, yang berarti bahwa ruleset ini akan langsung diterapkan ketika dibuat (ketika klik button <strong>Create</strong>). Kamu juga bisa membiarkannya default, yaitu <strong>Disabled</strong>, yang berarti bahwa ruleset ini belum akan diterapkan dan ketika nanti kamu ingin menerapkan ruleset ini, kamu bisa mengubah <strong>Enforcement Status</strong>-nya.</li>
<li>Pada bagian <strong>Bypass list</strong>, kamu bisa memberikan izin kepada <em>rules</em>, <em>teams</em> atau <em>apps</em> tertentu untuk <em>bypass</em> (mengabaikan atau melewati) ruleset ini. Tetapi karena dalam tutorial ini, ingin diterapkan bahwa tidak ada yang boleh mengabaikan ruleset ini, maka cukup dibiarkan kosong saja.</li>
<li>Pada bagian <strong>Target branches</strong>, kamu mendefinisikan di branch mana kamu ingin ruleset ini diterapkan. Mengacu pada konsep yang dijelaskan sebelumnya (bahwa tidak ada seorang pun yang dibolehkan melakukan perubahan langsung di branch main, dst), maka ruleset ini tentunya diterapkan pada branch main. Untuk itu klik <strong>Add target</strong> lalu pilih <strong>Include default branch</strong>, yang berarti bahwa ruleset ini akan diterapkan pada default branch, yaitu branch main (jika belum dilakukan perubahan).</li>
<li>Pada bagian <strong>Rules</strong>, kamu memilih <em>rule</em> apa saja yang ingin diterapkan. Secara default sudah ada dua <em>rule</em> yang otomatis dipilih oleh GitHub, yaitu rule pertama adalah <strong>Restrict deletions</strong>, yang berarti hanya user dengan izin <em>bypass</em> yang boleh menghapus branch yang didefinisikan pada bagian <strong>Target branches</strong>, yaitu dalam tutorial ini adalah branch main dan karena sebelumnya pada bagian <strong>Bypass list</strong> hanya dibiarkan kosong, maka artinya tidak ada seorang pun yang bisa menghapus branch main, ini bagus, karena jika branch main tidak sengaja terhapus, maka akan berakibat fatal. Yang kedua adalah <strong>Block force pushes</strong>, ini berarti bahwa tidak ada user yang diizinkan untuk melakukan <em>force push</em>.</li>
<li>Pilih rule <strong>Require pull request before merging</strong>, supaya jika ingin melakukan perubahan harus melalui pull request.</li>
<li>Pilih rule <strong>Require status checks to pass</strong>, lalu klik <strong>Add checks</strong>, pada kolom pencarian tulis id dari <em>job</em> yang terdapat pada workflow yang sebelumnya dibuat, sebagai contoh, id dari <em>job</em> yang saya buat sebelumnya adalah "test-and-lint" maka masukkan "test-and-lint" atau cukup "test" saja, lalu pada bagian <strong>Suggestions</strong> pilih <strong>Add test-and-lint</strong> dan klik kembali button <strong>Add checks</strong>:
<img src="/posts/deploy-vercel-github-action/require-status-checks-pass.png" alt="require status checks pass" width="1351" height="656" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=1351&height=656&loading=lazy&class=mt-6&decoding=async--></li>
<li>Berikut adalah tampilan akhir dari <em>settings ruleset</em>:
<img src="/posts/deploy-vercel-github-action/settings-ruleset.png" alt="settings ruleset" width="1366" height="2731" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=1366&height=2731&loading=lazy&class=mt-6&decoding=async--></li>
<li>Klik button <strong>Create</strong> untuk membuat ruleset.</li>
</ol>
<p>Setelah selesai membuat Branch Ruleset, saatnya melakukan test apakah ruleset tersebut berjalan sebagaimana mestinya. Kamu bisa mencoba membuat perubahan langsung pada branch main di local repositori, lalu coba lakukan push, maka kamu akan mendapatkan error seperti di bawah ini:
<img src="/posts/deploy-vercel-github-action/error-directly-change-main-branch.png" alt="error directly change main branch" width="1140" height="534" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=1140&height=534&loading=lazy&class=mt-6&decoding=async-->
Yang menandakan bahwa ruleset telah berjalan sebagaimana mestinya. Kamu bisa membatalkan perubahan sebelumnya dengan menjalankan command:</p>
<pre><code class="language-bash">git reset --hard HEAD~1
</code></pre>
<blockquote>
<p><strong>Info:</strong> Jika kamu belum mengetahui mengenai git-reset, bisa baca <a href="https://git-scm.com/docs/git-reset">disini</a>.</p>
</blockquote>
<p>Selain test dengan membuat perubahan langsung pada branch main, kamu juga bisa membuka pull request dengan sengaja membuat CI test menjadi error dan kamu bisa melihat bahwa button <strong>Merge pull request</strong> tidak bisa diklik, baik ketika <em>check</em> masih berjalan ataupun ketika <em>check</em> sudah selesai dan ada <em>check</em> yang error. Ketika ada <em>check</em> yang error seperti ini, kamu bisa melakukan update pada local repositori untuk memperbaiki error, lalu lakukan push dan proses CI akan dijalankan kembali. Ketika semua <em>check</em> lolos, maka kamu bisa melakukan <strong>Merge pull request</strong>.</p>
