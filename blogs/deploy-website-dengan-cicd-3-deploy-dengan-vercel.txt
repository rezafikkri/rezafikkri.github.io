10:T29db,<p>Setelah membuat GitHub Actions workflow pada seri ke-2, pada seri ke-3 ini kita akan sama-sama belajar deployment dengan menggunakan Vercel untuk proses CD.</p>
<blockquote>
<p><strong>Info:</strong> Pastikan kamu sudah mengikuti bagian ke-2. Jika belum, kamu bisa scroll ke paling bawah halaman ini sampai pada bagian Daftar Seri Tulisan dan klik link pada bagian ke-2.</p>
</blockquote>
<blockquote>
<p><strong>Info:</strong> Disini saya menggunakan akun GitHub lain (amruk53sh) untuk menunjukkan bagaimana cara deploy dengan Vercel untuk pertama kalinya.</p>
</blockquote>
<p>Untuk deploy dengan vercel ikuti langkah-langkah berikut:</p>
<ol>
<li>Buat akun Vercel dengan buka halaman <a href="https://vercel.com/signup">Sign Up Vercel</a>, lalu pada bagian <strong>Plan type</strong>, karena ini tujuannya untuk belajar maka pilih <strong>Hobby</strong>, lalu pada kolom input <strong>Your name</strong>, isi dengan nama kamu dan klik button <strong>Continue</strong>:
<img src="https://res.cloudinary.com/rezafikkri/image/upload/q_auto/sign-up-vercel.png" alt="sign up vercel" width="1351" height="656" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=1351&height=656&loading=lazy&class=mt-6&decoding=async--></li>
<li>Kamu akan diarahkan ke suatu halaman untuk menghubungkan akun Vercel kamu dengan Git provider, karena dalam tutorial ini menggunakan GitHub, maka klik button <strong>Continue With GitHub</strong>:
<img src="https://res.cloudinary.com/rezafikkri/image/upload/q_auto/connect-vercel-github.png" alt="connect vercel github" width="1351" height="656" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=1351&height=656&loading=lazy&class=mt-6&decoding=async--></li>
<li>Akan terbuka pop up window <em>Authorize Vercel</em>, yang intinya Vercel ingin meminta persetujuan dari kamu untuk mengakses beberapa hal di akun GitHub kamu. Klik button <strong>Authorize Vercel</strong> untuk menyetujui:
<img src="https://res.cloudinary.com/rezafikkri/image/upload/q_auto/authorize-vercel.png" alt="authorize vercel" width="818" height="680" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=818&height=680&loading=lazy&class=mt-6&decoding=async--></li>
<li>Kamu akan diminta mengisi nomor telepon. Pilih negara Indonesia, lalu isi dengan nomor aktif kamu (ini hanya untuk verifikasi) dan klik button <strong>Continue</strong>:
<img src="https://res.cloudinary.com/rezafikkri/image/upload/q_auto/insert-phone-number-vercel.png" alt="insert phone number vercel" width="826" height="682" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=826&height=682&loading=lazy&class=mt-6&decoding=async--></li>
<li>Kamu perlu memasukkan 4 kode yang telah dikirim ke nomor telepon kamu dan selamat akun Vercel kamu sudah berhasil dibuat:
<img src="https://res.cloudinary.com/rezafikkri/image/upload/q_auto/verify-vercel-sign-up.png" alt="verify vercel sign up" width="820" height="685" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=820&height=685&loading=lazy&class=mt-6&decoding=async--></li>
<li>Tunggu sebentar sampai kamu diarahkan ke halaman untuk membuat projek baru (seperti dibawah ini), jika agak lama, tidak apa-apa, tetap tunggu. Karena sebelumnya sudah memiliki projek yang ingin dideploy, maka kamu hanya perlu untuk melakukan <em>Import Git Repository</em> saja. Untuk melakukan import, kamu perlu install Vercel GitHub App di akun GitHub-mu, caranya dengan klik button <strong>Install</strong> pada bagian <strong>Import Git Repository</strong>:
<img src="https://res.cloudinary.com/rezafikkri/image/upload/q_auto/install-vercel.png" alt="install github app vercel" width="1366" height="1151" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=1366&height=1151&loading=lazy&class=mt-6&decoding=async--> Di GitHub ada tools yang bernama GitHub Apps, GitHub Apps adalah tools yang memperluas fungsionalitas GitHub. GitHub Apps dapat melakukan sesuatu di GitHub seperti membuka <em>issues</em>, komen di dalam pull request dan mengelola <em>projects</em>. GitHub Apps juga bisa melakukan sesuatu di luar GitHub berdasarkan <em>event</em> yang terjadi di GitHub. Vercel yang kamu install di akun GitHub-mu adalah salah satu dari GitHub Apps. Lebih detail mengenai GitHub Apps bisa baca <a href="https://docs.github.com/en/apps/using-github-apps/about-using-github-apps">disini</a>.</li>
<li>Kamu akan dimintai semacam persetujuan oleh Vercel untuk beberapa izin terkait menginstall Vercel GitHub App di akun GitHub-mu, kamu bisa membaca apa saja izin tersebut, selanjutnya jangan lupa untuk memilih <strong>Only select repositories</strong>, untuk install Vercel hanya di repositori yang dipilih saja, kemudian pilih repositori website counter-js dan klik button <strong>Install</strong> (scroll ke bawah jika kamu belum melihat button tersebut):
<img src="https://res.cloudinary.com/rezafikkri/image/upload/q_auto/permission-install-vercel.png" alt="permission install github app vercel" width="818" height="682" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=818&height=682&loading=lazy&class=mt-6&decoding=async--></li>
<li>Klik button <strong>Import</strong> pada repositori website counter-js di bagian <strong>Import Git Repository</strong>:
<img src="https://res.cloudinary.com/rezafikkri/image/upload/q_auto/importing-project-counter-js.png" alt="importing project counter js" width="1351" height="656" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=1351&height=656&loading=lazy&class=mt-6&decoding=async--></li>
<li>Sebelum deploy, kamu akan diminta untuk melakukan konfigurasi projek, seperti <em>Project Name</em>, <em>Framework Preset</em>, <em>Root Directory</em>, <em>Build and Output Settings</em> dan <em>Environment Variables</em>. Tetapi untuk di tutorial ini, kamu tidak perlu merubah apapun karena Vercel sudah membuat konfigurasi default, kamu bisa langsung klik button <strong>Deploy</strong> untuk deploy website counter-js dan tunggu sampai proses deploy selesai:
<img src="https://res.cloudinary.com/rezafikkri/image/upload/q_auto/configure-project-vercel.png" alt="configure project vercel" width="1351" height="656" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=1351&height=656&loading=lazy&class=mt-6&decoding=async--></li>
<li>Selamat kamu telah berhasil deploy website counter-js dengan Vercel. Selanjutnya kamu bisa klik button <strong>Continue to Dashboard</strong> untuk melihat detail dari website counter-js yang telah dideploy:
<img src="https://res.cloudinary.com/rezafikkri/image/upload/q_auto/success-deploy.png" alt="success deploy" width="1351" height="656" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=1351&height=656&loading=lazy&class=mt-6&decoding=async--></li>
<li>Kamu akan dialihkan ke halaman dashboard dari website counter-js yang telah berhasil dideploy. Untuk mengunjungi website yang telah dideploy kamu bisa klik button <strong>Visit</strong>:
<img src="https://res.cloudinary.com/rezafikkri/image/upload/q_auto/dashboard-vercel.png" alt="dashboard vercel" width="1366" height="759" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=1366&height=759&loading=lazy&class=mt-6&decoding=async--></li>
</ol>
<p>Setelah berhasil deploy website counter-js ke Vercel, seperti penjelasan saya sebelumnya, bahwa ketika ada perubahan yang diterapkan ke branch main, maka Vercel secara otomatis akan melakukan proses deployment. Untuk memastikan apakah betul berjalan seperti itu, buat branch baru, lalu lakukan perubahan apapun di repositori local (agar mudah melihat perbedaan antara deployment sebelumnya dan yang baru, buat perubahan pada tampilan website di file <code>main.js</code>) dan lakukan commit kemudian push.</p>
<blockquote>
<p><strong>Info:</strong> Disini saya kembali menggunakan akun GitHub utama saya (rezafikkri).</p>
</blockquote>
<p>Selanjutnya buka sebuah pull request ke branch main, tunggu semua pengecekan lolos dan lakukan <em>Merge pull request</em> dengan klik button <strong>Merge pull request</strong>:
<img src="https://res.cloudinary.com/rezafikkri/image/upload/q_auto/merge-pull-request.png" alt="merge pull request" width="1351" height="656" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=1351&height=656&loading=lazy&class=mt-6&decoding=async-->
Karena melakukan <em>merge</em>, maka kamu akan diminta untuk mengisi commit message dan commit description, kamu bisa merubahnya atau biarkan default dan klik button <strong>Confirm merge</strong>:
<img src="https://res.cloudinary.com/rezafikkri/image/upload/q_auto/confirm-merge-pull-request.png" alt="confirm merge pull request" width="1351" height="656" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=1351&height=656&loading=lazy&class=mt-6&decoding=async-->
Jika pada button tidak terdapat kata <em>Merge pull request</em>, maka klik dropdown menu pada button dan pilih <strong>Create a merge commit</strong>:
<img src="https://res.cloudinary.com/rezafikkri/image/upload/q_auto/button-merge-pull-request.png" alt="button merge pull request" width="1351" height="656" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=1351&height=656&loading=lazy&class=mt-6&decoding=async--></p>
<p>Setelah <em>Merge pull request</em> berhasil, untuk melihat apakah deployment otomatis dijalankan, opsi pertama, kamu bisa melihat langsung pada website counter-js yang telah dideploy, apakah terdapat perubahan (jika kamu melakukan perubahan pada tampilan website).</p>
<p>Opsi kedua, kamu bisa membuka tab <strong>Deployments</strong> pada halaman dashboard website counter-js, disana kamu bisa melihat deployment terbaru adalah deployment dari <em>Merge pull request</em> yang sebelumnya kamu lakukan, ini menandakan bahwa betul Vercel secara otomatis melakukan deployment ketika ada perubahan yang diterapkan pada branch main:
<img src="https://res.cloudinary.com/rezafikkri/image/upload/q_auto/automatic-deployment.png" alt="automatic deployment" width="1351" height="656" loading="lazy" class="mt-6" decoding="async"><!--rehype:width=1351&height=656&loading=lazy&class=mt-6&decoding=async--></p>
